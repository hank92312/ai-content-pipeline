# 🤖 網頁版 AI 模型腳本生成提示詞模板 (Web AI Prompt Templates)

本文件提供「小花的AI情報站」專屬提示詞模板。當您想使用網頁版 AI 模型（如 ChatGPT、Claude 或 Gemini 網頁版）來根據新聞生成本地系統可以讀取的 `.json` 腳本時，可以複製以下提示詞使用。

## 📋 模式一：標準 30 秒短影音模板

* **適用範疇**：3C、遊戲、動漫、新科技等娛樂/硬體情報。
* **特色**：熱情激動、資深玩家語氣，節奏明快（總字數約 100~120 字，包含 2 張配圖提示詞）。

### 提示詞內容（複製下方區塊）

```markdown

【核心人設與寫作規則】
1. 角色性別：蔡小花為「男性」（口吻請符合男性動漫宅/科技愛好者，避免使用偏女性化的語氣助詞或自稱）。
2. 外型風格：二次元科技風，具備反差萌。
3. 播報語氣：熱情、激動、像個資深玩家，口語流暢、幽默、自帶吐槽，絕對要像真人在說話，不要生硬地唸新聞稿。
4. 字數控制：腳本總字數（intro + main_content + outro）必須嚴格控制在 100 到 120 個繁體中文字之間，以符合 30 秒短影音節奏。
5. 腳本結構：
   - 開場白 (intro)：不要用死板的「歡迎回到...」。請根據新聞最勁爆的點，動態生成「三秒內抓住眼球」的破題開場。
   - 主內容 (main_content)：將新聞重點以講故事方式口語化帶過，字數不超過 60 字。
   - 收尾句 (outro)：拋出一個好玩的問題引導觀眾留言互動，並自然呼籲訂閱（例如：「留言區戰起來！記得訂閱小花，AI 科技情報不漏接喔！」）。

【配圖與關鍵字規則】
1. 必須在 `visual_prompts` 提供剛好 2 個英文生圖提示詞：
   - 格式必須包含 "2D anime style, visually stunning" 並與新聞內容相關。
   - 結尾必須加上 "No text, no letters, no words" 以防生圖出現亂碼。
   - 畫面一：針對新聞事件本身（開場重點）的畫面設計。
   - 畫面二：針對後續發展、或呼喚觀眾情緒的相關畫面設計。
2. 必須在 `keywords` 提供 2 到 4 個從文案中擷取的「高潮重點字或詞彙」，這些字會在影片中高亮顯示，請挑選最具情緒或最勁爆的詞。

## 🛠️ 本地使用與自動導入流程

1. **複製與生成**：選擇適合的模式提示詞，貼給網頁版 AI，填入新聞標題及內容，取得產出的 JSON。
2. **存入 source/ 資料夾**：
   在專案的 `source/` 資料夾下，新增一個 `.json` 檔案（例如 `source/news_temp.json`），並將 AI 生成的 JSON 內容貼入存檔。
3. **執行 step1_local_loader.py**：
   開啟終端機並執行：

   ```powershell
   python step1_local_loader.py
   ```

4. **互動確認**：
   * 程式偵測到 `source/` 下有該 JSON 檔。
   * 輸入分類選擇（1: Finance，2: Gaming）。
   * 程式將自動載入至 SQLite 資料庫，將狀態標記為 `is_processed = 1`。
   * 自動在 `output_scripts/` 產生對應的 `.txt` 與 `.json` 腳本檔案。
5. **接續語音合成**：
   程式會提示是否接續執行 `step3_voice_renderer.py`，輸入 `y` 即可自動跳過前面的 AI 生成步驟，直接進行配音渲染！
